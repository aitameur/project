## Project Overview

This is a PFA (Projet de Fin d'Année) repository for an OWASP Top 10 Web Vulnerability Scanner. 

**Current Phase**: Final report writing and documentation (not technical development)

## Scanner Architecture (`imple/`) [COMPLETED - FOR REFERENCE ONLY]

The scanner follows a modular pipeline architecture:

```
Crawler → Detectors → CVSSEvaluator → HTMLReporter
```

### Core Components

- **`main.py`**: CLI entry point using argparse. Handles all command-line options including DVWA auto-login.
- **`scanner/core.py`**: Orchestrator (`Scanner` class) that coordinates the full pipeline: crawl → detect → evaluate → report.
- **`scanner/vulnerability.py`**: Core `Vulnerability` dataclass representing detected issues.
- **`scanner/authenticator.py`**: Auto-login mechanisms for DVWA and generic form-based auth.
- **`crawler/crawler.py`**: BFS-based web crawler that discovers endpoints and forms. Returns `Endpoint` objects with URL, method, and injectable parameters.
- **`detectors/`**: Specialized detector classes for each vulnerability type. Each inherits from `BaseDetector` or `PassiveDetector`.
- **`evaluator/cvss.py`**: CVSS v3.1 scoring engine (`CVSSEvaluator`).
- **`reporter/html_reporter.py`**: Generates standalone HTML reports with embedded CSS/JS.
- **`payloads/`**: Text files containing attack payloads (SQLi, XSS, SSRF, Path Traversal, XXE).

### Detector Architecture

**Active Detectors** (inherit from `BaseDetector`):
- Scan endpoints with injectable parameters (forms, query strings).
- Inject payloads from `payloads/*.txt` files.
- Examples: `SQLiDetector`, `XSSDetector`, `SSRFDetector`, `IDORDetector`, `PathTraversalDetector`, `XXEDetector`, `FileUploadDetector`.

**Passive Detectors** (inherit from `PassiveDetector`):
- Analyze all crawled pages without sending payloads.
- Check HTTP headers, response content, and page metadata.
- Examples: `CryptographicFailuresDetector`, `SecurityMisconfigDetector`, `VulnerableComponentsDetector`, `AuthFailuresDetector`, `IntegrityDetector`.

All detectors registered in `detectors/__init__.py` in the `ALL_DETECTORS` dictionary.

### Import Structure

Due to circular import avoidance, `Scanner` is imported lazily:
```python
from scanner import Scanner          # Lazy import via __getattr__
from scanner.core import Scanner     # Direct import
```

`Vulnerability` is available directly:
```python
from scanner import Vulnerability
```

## Scanner Development Commands [FOR REFERENCE ONLY]

### Setup

```bash
cd imple
python -m venv .venv

# Windows
.venv\Scripts\activate

# Linux/macOS
source .venv/bin/activate

pip install -r requirements.txt
```

**Requirements**: Python 3.10+

### Running the Scanner

```bash
# Full scan
python main.py --url http://127.0.0.1:5000

# Targeted scan (specific detectors)
python main.py --url http://127.0.0.1:5000 --only sqli,xss

# DVWA auto-login + scan
python main.py --url http://127.0.0.1/dvwa --dvwa --dvwa-security low

# Advanced options
python main.py --url http://target --depth 3 --timeout 15 --max-pages 200 --output custom_report.html -v
```

### Available Detectors

Use with `--only` flag (comma-separated):
- `sqli` - SQL Injection (A03)
- `xss` - Cross-Site Scripting (A03)
- `ssrf` - Server-Side Request Forgery (A10)
- `idor` - Insecure Direct Object Reference (A01)
- `path_traversal` - Path Traversal (A01)
- `xxe` - XML External Entity (A05)
- `file_upload` - Unrestricted File Upload (A04)
- `crypto` - Cryptographic Failures (A02)
- `security_misconfig` - Security Misconfiguration (A05)
- `components` - Vulnerable Components (A06)
- `auth` - Authentication Failures (A07)
- `integrity` - Software & Data Integrity Failures (A08)

## Document Structure Reference

### Word Document Unpacking

Word `.docx` files are ZIP archives containing XML:
```
rapport.docx (unpacked as rapport_analysis_unpacked/)
├── word/
│   ├── document.xml          # Main content
│   ├── styles.xml            # Styling definitions
│   ├── media/                # Embedded images
│   └── _rels/
│       └── document.xml.rels # Image relationships
├── docProps/                 # Metadata
└── [Content_Types].xml       # MIME types
```

To manually unpack/repack:
```bash
# Unpack
unzip rapport.docx -d rapport_unpacked/

# Repack (after modifications)
cd rapport_unpacked/
zip -r ../rapport_modified.docx *
```
